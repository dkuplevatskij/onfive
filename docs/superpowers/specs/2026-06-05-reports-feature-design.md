# Дизайн фичи «Доклады» (OnFive)

**Дата:** 2026-06-05
**Статус:** утверждён, готов к планированию реализации
**Фаза:** 2 (Доклады + голос)

## Цель

Добавить в OnFive раздел «Доклады» — AI-помощник для написания школьных
докладов учениками 5–11 классов. Сохраняет философию OnFive: AI **не пишет
доклад за ученика** (кроме явного режима «черновик»), а ведёт его сократическим
методом к самостоятельному результату. Готовый доклад экспортируется в `.docx`
и `.pdf` с подписью «Подготовлено с помощью OnFive AI».

## Ключевые решения (из брейнсторма)

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Размещение в навигации | Карточка «📝 Доклады» на Главной + отдельный флоу. Нижние вкладки не трогаем. |
| 2 | Хранение черновиков | `localStorage` через новый Zustand-store `useReportsStore`. Без бэкенда, офлайн. Миграция в Supabase — позже. |
| 3 | Экспорт `.docx`/`.pdf` | Клиентский JS: npm `docx` для Word, `jspdf` + встроенный кириллический шрифт для PDF. |
| 4 | AI-флоу | Переиспользуем `/api/chat` с новыми системными промптами. Режимы доклада хранятся **отдельно** от 5 учебных. |
| 5 | Рабочее пространство | Две вкладки внутри экрана доклада: «💬 Чат» и «📄 Текст». |
| 6 | Голос/фото в докладах | НЕ делаем в этой итерации (упрощение). |

## Архитектура

### Роуты (в `PlainLayout`, без вкладок)

| Роут | Страница | Назначение |
|------|----------|-----------|
| `/reports` | `ReportsList` | Список моих докладов + кнопка «Новый доклад» |
| `/reports/new` | `ReportNew` | Мастер: предмет → тема → объём → режим |
| `/reports/:id` | `ReportWorkspace` | Рабочее пространство (вкладки Чат / Текст) |

Вход — карточка «📝 Доклады» на `Home.tsx`, ведёт на `/reports`.

### Хранение: `useReportsStore` (Zustand + persist, ключ `onfive-reports`)

```ts
type ReportLength = "short" | "medium" | "long";   // ~500 / 1000 / 2000 слов
type ReportMode = "write" | "draft" | "review";    // помоги / черновик / проверь

interface Report {
  id: string;                // crypto.randomUUID()
  subject: SubjectId;
  topic: string;             // тема доклада
  length: ReportLength;
  mode: ReportMode;
  messages: ChatMessage[];   // история чата с AI
  draft: string;             // накопленный текст доклада (markdown)
  createdAt: string;         // ISO
  updatedAt: string;         // ISO
}

interface ReportsState {
  reports: Report[];
  create(input: { subject; topic; length; mode }): string;  // → id
  get(id: string): Report | undefined;
  update(id: string, patch: Partial<Report>): void;
  appendToDraft(id: string, text: string): void;            // «В доклад»
  remove(id: string): void;
}
```

### AI-флоу: переиспользование `/api/chat`

Расширяем `ChatContext` (в `shared/src/types.ts`) необязательными полями для
докладов — режимы доклада **не смешиваются** с 5 учебными `LearningMode`:

```ts
interface ChatContext {
  grade: Grade;
  subject: SubjectId;
  topic: string;
  mode: LearningMode;              // для докладов передаём "free" как базовый
  goals?: string[];
  reportMode?: ReportMode;         // если задан — это сессия доклада
  reportLength?: ReportLength;
}
```

В `api/chat.ts` (и зеркально в `server/src/services/prompts.ts`) при наличии
`reportMode` системный промпт строится из шаблонов докладов вместо учебных.

**Промпты (из CLAUDE.md):**

- **write** (сократический): уточни тему и объём → предложи план (введение,
  3–5 пунктов, заключение) → по каждому пункту задавай ученику вопросы → на
  основе ответов помогай сформулировать абзац. НЕ писать доклад целиком сам.
- **draft**: создай структурированный черновик заданного объёма (`reportLength`),
  язык на уровне класса, структура Введение → Основная часть (3–5 разделов) →
  Заключение → Список источников. В конце предложи прочитать и задать вопросы.
- **review**: ученик вставляет свой текст → дай конструктивную обратную связь
  (структура, аргументация, грамотность), но НЕ переписывай за него.

Все промпты сохраняют базовое правило OnFive (уровень класса, обращение на «ты»,
формулы в LaTeX, мотивация).

### Рабочее пространство `/reports/:id` (`ReportWorkspace`)

- Шапка: тема доклада + чип режима/объёма.
- Переключатель вкладок вверху: **💬 Чат** | **📄 Текст**.
- **Вкладка Чат:** переиспользует UI из `Chat.tsx` (компонент `Markdown`,
  пузыри сообщений, поле ввода). Голос/фото отключены. Под каждым ответом AI —
  кнопка **«➕ В доклад»**, добавляющая текст ответа в `draft` (`appendToDraft`).
- **Вкладка Текст:** редактируемая `<textarea>` с markdown-черновиком +
  кнопки **«⬇️ Скачать .docx»** и **«⬇️ Скачать .pdf»**.

### Экспорт (клиент)

- `lib/export/docx.ts` — через npm `docx`: заголовок = тема, тело = абзацы из
  markdown-черновика, футер «Подготовлено с помощью OnFive AI». Скачивание через
  Blob.
- `lib/export/pdf.ts` — через `jspdf`. Встраиваем кириллический шрифт (PT Sans,
  base64 в `lib/export/ptSansFont.ts`), т.к. дефолтные шрифты jsPDF не
  поддерживают кириллицу. Та же подпись в футере.
- Имя файла: `<тема>-onfive.docx` / `.pdf` (транслит/очистка спецсимволов).

### Геймификация

- Достижения **«Автор»** (первый доклад) и **«Докладчик»** (10 докладов) уже
  определены в `data/achievements.ts`. Число докладов для условия ачивок берём
  из `useReportsStore.reports.length` — там же, где вычисляются остальные ачивки.
- XP начисляем через новый метод `recordReport()` в `useUserStore` (по аналогии
  с `recordLesson`): **+15** при создании доклада. Метод вызывается из
  `ReportNew` после `create`. Экспорт отдельным XP не награждаем (упрощение).

## Тестирование

- **Юнит:** `useReportsStore` (create/update/appendToDraft/remove, персист).
- **Юнит:** построение системного промпта для каждого `reportMode`.
- **Юнит:** экспорт — docx/pdf генерируют непустой Blob с подписью OnFive;
  кириллица в PDF не теряется (проверка наличия шрифта).
- **Ручной (browser-qa):** полный путь Главная → Доклады → новый → чат → план →
  «В доклад» → редактирование → экспорт обоих форматов.

## Вне области (YAGNI для этой итерации)

- Голосовой ввод и озвучка в докладах.
- Синхронизация докладов через Supabase (отложено до фазы аккаунтов).
- Серверная генерация документов.
- Шаринг доклада в Telegram (фаза 3).
- Структурированный JSON-план как отдельная сущность — план живёт в чате как текст.
